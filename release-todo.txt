Steps for releasing a new CGDB version:

1.  Update all ChangeLogs to have the new release date and version.
    The ChangeLogs live in many places currently.
    find . -name ChangeLog

2.  Update the NEWS file to have the new release date and version.
    
3.  Change configure.in file to reflect new version number.
      * Run autoregen.sh
    Make sure you have the appropriate versions of these installed:
      * aclocal (GNU automake) 1.7
      * autoconf (GNU Autoconf) 2.57
      * libtool ( GNU libtool ) 1.4.3
      * m4      ( GNU m4 )     1.4

	Also, change tgdb/Doxyfile and cgdb/Doxyfile to have the correct
	version number. You want to change the PROJECT_NUMBER line.
      
4.  Test doing 'make dist' then untar, ./configure && make
    and run ./cgdb --version to check version number.

5.  Create a cvs tag for the tree to be able to get back to the 
    version number. If the version is cgdb-0.3.3, name the tag
	cgdb-0_3_3 .

6.  Put a release on sourceforge.net/projects/cgdb
    * ftp upload.sourceforge.net
      * user=anonymous, passwd=email
      * type bin
      * cd incoming
      * put the file up
    * go to sourceforge.net/projects/cgdb
    * go to admin, then edit/release files
	* you should be able to put whats new in the NEWS file into the
	  release notes.
	* Copy new README file into readme.txt

7.  Download and test the new version from sourceforge.

8.  Update website cgdb.sourceforge.net:
    * ssh cgdb.sourceforge.net
    * cd /home/groups/c/cg/cgdb/htdocs
    * Modify download page -> download.shtml
      * Download link
      * Changelog link (point to the correct version in CVS)
    * Optionally, put a note in the index.shtml file (News section)

9.  Test website by looking at it and attempting to download file.
    
10.  Announce the release on freshmeat

11.  Announce the release on cgdb-announce@lists.sourceforge.net

Subject:  CGDB (version) released
Text of message:

CGDB (version) Released
-------------------

Downloading:

    Go to http://cgdb.sourceforge.net/download.shtml for download
    link and instructions.

This new version contains the following changes:

    * Changelog entry 1
    * Changelog entry 2
      (no long descriptions, just the part before the ':')
